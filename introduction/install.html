Adım 1

Sistem Gereksinimleri

RPD App'i çalıştırmak için aşağıdaki araçların sisteminizde kurulu olması gerekmektedir.

Node.js
v20.0.0 veya üzeri

JavaScript runtime ortamı. Next.js ve tüm bağımlılıkları çalıştırmak için gereklidir. LTS (Long Term Support) sürümünü tercih edin.

Node.js İndir
npm / yarn / pnpm
npm v10+ önerilir

Paket yöneticisi. Node.js ile birlikte npm otomatik kurulur. Alternatif olarak yarn veya pnpm kullanabilirsiniz.

npm Dokümantasyon
Git
v2.40+ önerilir

Versiyon kontrol sistemi. Projeyi GitHub'dan klonlamak ve güncellemeleri almak için gereklidir.

Git İndir
VS Code (Önerilen)
Son sürüm

Kod editörü. TypeScript, Tailwind CSS ve Next.js için mükemmel destek sağlar. Önerilen eklentiler: ESLint, Prettier, Tailwind CSS IntelliSense.

VS Code İndir
Sürümleri Kontrol Edin
Kurulumlarınızı doğrulamak için terminalde aşağıdaki komutları çalıştırın:
Terminal 
# Node.js sürümünü kontrol et
node --version
# Beklenen: v20.x.x veya üzeri

# npm sürümünü kontrol et
npm --version
# Beklenen: 10.x.x veya üzeri

# Git sürümünü kontrol et
git --version
# Beklenen: git version 2.40.x veya üzeri
Adım 2

Proje Kurulumu

Projeyi GitHub'dan klonlayın ve bağımlılıkları yükleyin.

2.1 Projeyi Klonlayın

Terminal veya komut satırını açın ve aşağıdaki komutu çalıştırın:

Terminal 
# Projeyi klonla
git clone https://github.com/mervedalgin/rpd-app.git

# Proje dizinine gir
cd rpd-app

2.2 Bağımlılıkları Yükleyin

Proje dizininde aşağıdaki komutlardan birini çalıştırın:

Terminal - npm ile 
# npm ile bağımlılıkları yükle
npm install

# veya yarn ile
yarn install

# veya pnpm ile
pnpm install
Kurulum Tamamlandı!
Bağımlılıklar başarıyla yüklendiyse, node_modules klasörü oluşmuş olmalıdır. Şimdi environment değişkenlerini yapılandırmaya geçebilirsiniz.

2.3 Proje Yapısı

Projenin temel dosya yapısı aşağıdaki gibidir:

Proje Yapısı 
rpd-app/
├── src/
│   ├── app/                 # Next.js App Router
│   │   ├── api/            # API endpoints
│   │   ├── panel/          # Panel sayfaları
│   │   ├── globals.css     # Global stiller
│   │   ├── layout.tsx      # Ana layout
│   │   └── page.tsx        # Ana sayfa
│   ├── components/         # React componentleri
│   │   ├── ui/            # shadcn/ui bileşenleri
│   │   ├── mebbis/        # MEBBİS bileşenleri
│   │   └── charts/        # Grafik bileşenleri
│   ├── lib/               # Utility fonksiyonları
│   └── types/             # TypeScript tip tanımları
├── public/                # Statik dosyalar
├── supabase/
│   └── migrations/        # Veritabanı migration dosyaları
├── var/                   # Veri dosyaları (teachers.json)
├── .env.local             # Environment değişkenleri (oluşturulacak)
├── package.json           # Proje bağımlılıkları
└── next.config.ts         # Next.js yapılandırması
Adım 3

Environment Değişkenleri

Uygulama yapılandırması için gerekli ortam değişkenlerini ayarlayın.

3.1 .env.local Dosyasını Oluşturun

Proje kök dizininde .env.local adında bir dosya oluşturun:

Terminal 
# Windows
copy NUL .env.local

# macOS / Linux
touch .env.local

3.2 Tüm Environment Değişkenleri

Aşağıdaki tüm değişkenleri .env.local dosyasına ekleyin ve kendi değerlerinizle doldurun:

.env.local 
# ===========================================
# SUPABASE YAPILANDIRMASI
# ===========================================
# Supabase projenizin URL'i
NEXT_PUBLIC_SUPABASE_URL=https://xxxxxxxxxxxxx.supabase.co

# Supabase anon (public) key
NEXT_PUBLIC_SUPABASE_ANON_KEY=eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9...

# Supabase service role key (sunucu tarafı için)
SUPABASE_SERVICE_ROLE_KEY=eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9...

# ===========================================
# TELEGRAM BOT YAPILANDIRMASI
# ===========================================
# Telegram Bot Token (@BotFather'dan alınır)
TELEGRAM_BOT_TOKEN=7123456789:AAHdqTcvCH1vGWJxfSeofSAs0K5PALDsaw

# Telegram Chat ID (grup veya kanal ID'si)
TELEGRAM_CHAT_ID=-1001234567890

# ===========================================
# GOOGLE SHEETS API YAPILANDIRMASI
# ===========================================
# Google Sheets dosya ID'si
GOOGLE_SHEETS_ID=1BxiMVs0XRA5nFMdKvBdBZjgmUUqptlbs74OgvE2upms

# Service Account email adresi
GOOGLE_SERVICE_ACCOUNT_EMAIL=rpd-app@your-project.iam.gserviceaccount.com

# Service Account private key (JSON'dan alınır)
GOOGLE_PRIVATE_KEY="-----BEGIN PRIVATE KEY-----\nMIIEvQIBADANBgkqhk...\n-----END PRIVATE KEY-----\n"

# ===========================================
# UYGULAMA AYARLARI (Opsiyonel)
# ===========================================
# Uygulama URL'i (production için)
NEXT_PUBLIC_APP_URL=https://your-domain.vercel.app

# Cron job secret key
CRON_SECRET=your-random-secret-key-here
Güvenlik Uyarısı
.env.local dosyası hassas bilgiler içerir. Bu dosyayı asla Git'e commit etmeyin! Proje zaten .gitignore ile bu dosyayı hariç tutar.
Sonraki Adımlar
Her bir entegrasyon için gerekli değerleri nasıl alacağınız aşağıdaki bölümlerde detaylı olarak açıklanmıştır. Sırasıyla Supabase, Google Sheets ve Telegram kurulumunu yapın.
Adım 4

Supabase Kurulumu

Veritabanı ve backend servisleri için Supabase'i yapılandırın.

4.1 Supabase Hesabı Oluşturun

Supabase
Ücretsiz plan yeterli

1. supabase.com adresine gidin
2. "Start your project" butonuna tıklayın
3. GitHub hesabınızla giriş yapın
4. "New Project" ile yeni proje oluşturun
5. Proje adı girin ve güçlü bir veritabanı şifresi belirleyin
6. Region olarak eu-central-1 (Frankfurt) seçin

Supabase Dashboard

4.2 API Anahtarlarını Alın

Proje oluşturulduktan sonra, Settings → API bölümüne gidin:

API Anahtarları 
# Project URL - "Project URL" başlığı altında
NEXT_PUBLIC_SUPABASE_URL=https://xxxxxxxxxxxxx.supabase.co

# Anon/Public Key - "Project API keys" altında "anon public"
NEXT_PUBLIC_SUPABASE_ANON_KEY=eyJhbGciOi...

# Service Role Key - "Project API keys" altında "service_role" (⚠️ Gizli tutun!)
SUPABASE_SERVICE_ROLE_KEY=eyJhbGciOi...

4.3 Veritabanı Tablolarını Oluşturun

Supabase Dashboard'da SQL Editor'e gidin ve aşağıdaki migration'ları sırasıyla çalıştırın:

001_create_tables.sql 
-- Öğrenci tablosu
CREATE TABLE IF NOT EXISTS students (
    id UUID DEFAULT gen_random_uuid() PRIMARY KEY,
    ogrenci_no VARCHAR(20) UNIQUE NOT NULL,
    ad_soyad VARCHAR(100) NOT NULL,
    sinif VARCHAR(10) NOT NULL,
    sube VARCHAR(5) NOT NULL,
    cinsiyet VARCHAR(10),
    dogum_tarihi DATE,
    veli_adi VARCHAR(100),
    veli_telefon VARCHAR(20),
    adres TEXT,
    created_at TIMESTAMPTZ DEFAULT NOW(),
    updated_at TIMESTAMPTZ DEFAULT NOW()
);

-- Yönlendirme tablosu
CREATE TABLE IF NOT EXISTS referrals (
    id UUID DEFAULT gen_random_uuid() PRIMARY KEY,
    student_id UUID REFERENCES students(id),
    ogretmen_adi VARCHAR(100) NOT NULL,
    nedenler TEXT[] NOT NULL,
    notlar TEXT,
    durum VARCHAR(20) DEFAULT 'beklemede',
    created_at TIMESTAMPTZ DEFAULT NOW(),
    updated_at TIMESTAMPTZ DEFAULT NOW()
);

-- Öğretmen tablosu
CREATE TABLE IF NOT EXISTS teachers (
    id UUID DEFAULT gen_random_uuid() PRIMARY KEY,
    ad_soyad VARCHAR(100) NOT NULL,
    brans VARCHAR(100),
    sinif VARCHAR(10),
    telefon VARCHAR(20),
    email VARCHAR(100),
    created_at TIMESTAMPTZ DEFAULT NOW()
);

-- Indexler
CREATE INDEX IF NOT EXISTS idx_students_sinif ON students(sinif);
CREATE INDEX IF NOT EXISTS idx_students_ogrenci_no ON students(ogrenci_no);
CREATE INDEX IF NOT EXISTS idx_referrals_student ON referrals(student_id);
CREATE INDEX IF NOT EXISTS idx_referrals_durum ON referrals(durum);
002_create_appointments.sql 
-- Randevu tablosu
CREATE TABLE IF NOT EXISTS appointments (
    id UUID DEFAULT gen_random_uuid() PRIMARY KEY,
    student_id UUID REFERENCES students(id),
    tarih DATE NOT NULL,
    saat TIME NOT NULL,
    konu VARCHAR(200),
    notlar TEXT,
    durum VARCHAR(20) DEFAULT 'planlandı',
    hatirlatici_gonderildi BOOLEAN DEFAULT FALSE,
    created_at TIMESTAMPTZ DEFAULT NOW(),
    updated_at TIMESTAMPTZ DEFAULT NOW()
);

-- Randevu görevleri tablosu
CREATE TABLE IF NOT EXISTS appointment_tasks (
    id UUID DEFAULT gen_random_uuid() PRIMARY KEY,
    appointment_id UUID REFERENCES appointments(id) ON DELETE CASCADE,
    gorev TEXT NOT NULL,
    tamamlandi BOOLEAN DEFAULT FALSE,
    created_at TIMESTAMPTZ DEFAULT NOW()
);

-- Indexler
CREATE INDEX IF NOT EXISTS idx_appointments_tarih ON appointments(tarih);
CREATE INDEX IF NOT EXISTS idx_appointments_student ON appointments(student_id);
CREATE INDEX IF NOT EXISTS idx_appointments_durum ON appointments(durum);
Tablo Oluşturma İpucu
Migration dosyalarının tamamı supabase/migrations/ klasöründe bulunmaktadır. Her dosyayı sırasıyla SQL Editor'de çalıştırın veya Supabase CLI kullanın.
Adım 5

Google Sheets API

Öğrenci verilerini Google Sheets'ten çekmek için API yapılandırması.

5.1 Google Cloud Projesi Oluşturun

Google Cloud Console'da yeni bir proje oluşturun:

Google Cloud Console
Ücretsiz kullanım kotası mevcut

1. console.cloud.google.com adresine gidin
2. Üstteki proje seçiciden "New Project" seçin
3. Proje adı olarak "rpd-app" girin
4. "Create" butonuna tıklayın

Yeni Proje Oluştur

5.2 Google Sheets API'yi Etkinleştirin

Oluşturduğunuz projede API'yi aktif edin:

Adımlar 
1. APIs & Services → Library'ye gidin
2. "Google Sheets API" arayın
3. "Enable" butonuna tıklayın
4. API etkinleştirilene kadar bekleyin

5.3 Service Account Oluşturun

API'ye erişim için Service Account oluşturun:

Service Account Adımları 
1. APIs & Services → Credentials'a gidin
2. "+ CREATE CREDENTIALS" → "Service account" seçin
3. Service account adı: "rpd-sheets-access"
4. "Create and Continue" tıklayın
5. Role olarak "Editor" seçin (opsiyonel)
6. "Done" ile tamamlayın

5.4 JSON Key Dosyasını İndirin

Service Account için JSON key oluşturun:

Key Oluşturma 
1. Credentials sayfasında Service Account'ı tıklayın
2. "Keys" sekmesine gidin
3. "Add Key" → "Create new key"
4. Key type: "JSON" seçin
5. "Create" tıklayın - JSON dosyası indirilecek

5.5 JSON'dan Değerleri Alın

İndirilen JSON dosyasından aşağıdaki değerleri .env.local'e kopyalayın:

JSON → .env.local 
// İndirilen JSON dosyası şuna benzer:
{
  "type": "service_account",
  "project_id": "rpd-app-xxxxx",
  "private_key_id": "abc123...",
  "private_key": "-----BEGIN PRIVATE KEY-----\n...\n-----END PRIVATE KEY-----\n",
  "client_email": "rpd-sheets-access@rpd-app-xxxxx.iam.gserviceaccount.com",
  ...
}

// .env.local'e şu şekilde ekleyin:
// client_email → GOOGLE_SERVICE_ACCOUNT_EMAIL
// private_key → GOOGLE_PRIVATE_KEY (tırnak içinde, \n'ler korunmalı)

5.6 Google Sheets'e Erişim Verin

Son olarak, Service Account'a Sheets erişimi verin:

Sheets Paylaşım 
1. Google Sheets dosyanızı açın
2. Sağ üstteki "Share" butonuna tıklayın
3. Service Account email'ini yapıştırın:
   rpd-sheets-access@rpd-app-xxxxx.iam.gserviceaccount.com
4. "Editor" yetkisi verin
5. "Send" ile paylaşın

6. Sheets ID'yi URL'den alın:
   https://docs.google.com/spreadsheets/d/[SHEETS_ID]/edit
   ↑ Bu kısım GOOGLE_SHEETS_ID değerinizdir
Private Key Formatı
GOOGLE_PRIVATE_KEY değerini çift tırnak içinde yazın ve \n karakterlerini koruyun. Hatalı format API hatalarına neden olur.
Adım 6

Telegram Bot Kurulumu

Bildirim sistemi için Telegram Bot oluşturun ve yapılandırın.

6.1 Bot Oluşturun

Telegram'da @BotFather ile yeni bot oluşturun:

BotFather Komutları 
# 1. Telegram'da @BotFather'ı arayın ve başlatın

# 2. Yeni bot oluşturun
/newbot

# 3. Bot için isim girin (görünen ad)
RPD Bildirim Botu

# 4. Bot için kullanıcı adı girin (benzersiz, _bot ile bitmeli)
rpd_okul_bot

# 5. BotFather size token verecek:
# "Done! Congratulations on your new bot..."
# Use this token to access the HTTP API:
# 7123456789:AAHdqTcvCH1vGWJxfSeofSAs0K5PALDsaw
# ↑ Bu TELEGRAM_BOT_TOKEN değerinizdir

6.2 Grup/Kanal Oluşturun

Bildirimleri alacak grup veya kanalı oluşturun:

Grup Kurulumu 
1. Telegram'da yeni grup oluşturun: "RPD Bildirimleri"
2. Botu gruba ekleyin (@rpd_okul_bot)
3. Botu admin yapın (mesaj gönderebilmesi için):
   - Grup ayarları → Administrators → Add Admin
   - Botu seçin ve yetkileri verin

6.3 Chat ID'yi Öğrenin

Grubun Chat ID'sini bulmak için:

Chat ID Bulma 
# Yöntem 1: @userinfobot kullanın
# 1. @userinfobot'u gruba ekleyin
# 2. Bot otomatik olarak Chat ID'yi gösterecek
# 3. Sonra botu gruptan çıkarabilirsiniz

# Yöntem 2: API ile
# 1. Gruba herhangi bir mesaj gönderin
# 2. Tarayıcıda şu URL'yi açın:
https://api.telegram.org/bot[BOT_TOKEN]/getUpdates

# 3. JSON yanıtında "chat" → "id" değerini bulun:
# "chat": {"id": -1001234567890, "title": "RPD Bildirimleri", ...}
#          ↑ Bu TELEGRAM_CHAT_ID değerinizdir (eksi işareti dahil!)

6.4 Botu Test Edin

Her şeyin doğru çalıştığını test edin:

Test Mesajı 
# Tarayıcıda bu URL'yi açın (değerleri kendi bilgilerinizle değiştirin):
https://api.telegram.org/bot[BOT_TOKEN]/sendMessage?chat_id=[CHAT_ID]&text=Test%20mesajı

# Örnek:
https://api.telegram.org/bot7123456789:AAHdqTcvCH1vGWJxfSeofSAs0K5PALDsaw/sendMessage?chat_id=-1001234567890&text=RPD%20App%20bağlantısı%20başarılı!

# Başarılıysa gruba mesaj gelir ve JSON yanıtında "ok": true görürsünüz
Telegram Hazır!
Test mesajı grubunuza ulaştıysa, Telegram entegrasyonu tamamlanmış demektir. Artık uygulama üzerinden yapılan yönlendirmeler otomatik olarak gruba bildirilecek.

Deployment (Yayınlama)

Uygulamayı Vercel veya Docker ile production'a alın.

7.1 Vercel ile Deployment (Önerilen)

En kolay yol Vercel ile otomatik deployment:

Vercel
Next.js için optimize

1. vercel.com'a GitHub ile giriş yapın
2. "Add New Project" tıklayın
3. rpd-app repository'sini import edin
4. Environment Variables bölümüne .env.local değerlerini ekleyin
5. "Deploy" butonuna tıklayın

Vercel'de Deploy Et
Docker
Self-hosted için

Kendi sunucunuzda barındırmak istiyorsanız Docker kullanabilirsiniz. Projede hazır Dockerfile ve docker-compose.yml bulunmaktadır.

Docker İndir

7.2 Vercel Environment Variables

Vercel dashboard'da şu değişkenleri ekleyin:

Vercel → Settings → Environment Variables 
# Her değişkeni ayrı ayrı ekleyin:
NEXT_PUBLIC_SUPABASE_URL        = https://xxx.supabase.co
NEXT_PUBLIC_SUPABASE_ANON_KEY   = eyJhbGci...
SUPABASE_SERVICE_ROLE_KEY       = eyJhbGci...
TELEGRAM_BOT_TOKEN              = 7123456789:AAH...
TELEGRAM_CHAT_ID                = -1001234567890
GOOGLE_SHEETS_ID                = 1BxiMVs0XRA5nF...
GOOGLE_SERVICE_ACCOUNT_EMAIL    = xxx@xxx.iam.gserviceaccount.com
GOOGLE_PRIVATE_KEY              = "-----BEGIN PRIVATE KEY-----\n..."

7.3 Docker ile Deployment

Docker kullanarak deploy etmek için:

Docker Komutları 
# Docker image oluştur
docker build -t rpd-app .

# Container'ı başlat
docker run -p 3000:3000 --env-file .env.local rpd-app

# Veya docker-compose ile (önerilen)
docker-compose up -d

# Logları görüntüle
docker-compose logs -f

# Durdur
docker-compose down

Sorun Giderme

Sık karşılaşılan sorunlar ve çözümleri.

Node.js sürümünüzü kontrol edin. v20 veya üzeri gereklidir:

node --version

Ayrıca npm cache temizlemeyi deneyin:

npm cache clean --force
rm -rf node_modules package-lock.json
npm install

.env.local dosyasında şunları kontrol edin:

  • NEXT_PUBLIC_SUPABASE_URL https:// ile başlamalı
  • URL sonunda / olmamalı
  • Anahtarlar tam olarak kopyalanmış olmalı

Sunucuyu yeniden başlatın:

npm run dev

Kontrol edilecekler:

  • Bot token doğru mu?
  • Chat ID eksi (-) işaretiyle birlikte mi yazıldı?
  • Bot gruba admin olarak eklenmiş mi?
  • Bot'a "Send Messages" yetkisi verilmiş mi?

Test için tarayıcıda deneyin:

https://api.telegram.org/bot[TOKEN]/sendMessage?chat_id=[CHAT_ID]&text=Test

GOOGLE_PRIVATE_KEY formatı kritik:

  • Değer çift tırnak içinde olmalı: "-----BEGIN..."
  • \n karakterleri korunmalı, gerçek satır sonu olmamalı
  • Service Account email'i Sheets'e Editor olarak paylaşılmış olmalı

Önce .next klasörünü temizleyin:

npm run clean
npm run build

TypeScript hatası varsa:

npm run type-check

ESLint hatası varsa:

npm run lint:fix
